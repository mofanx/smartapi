## 1. 变量系统

SmartAPI-Test 提供五层变量体系，优先级从高到低：

| 层级 | 说明 | 设置方式 |
|------|------|---------|
| **步骤变量** | 当前步骤内有效 | `steps[].variables` |
| **用例变量** | 当前用例内有效 | `variables` (顶层) |
| **提取变量** | 从响应中提取，跨步骤共享 | `steps[].extract` |
| **全局变量** | 跨用例共享 | CLI `--var` 或 conftest |
| **环境变量** | 来自环境配置文件 | `environments/*.yaml` |

### 变量引用语法

```yaml
# 普通变量引用
url: "/api/users/${user_id}"

# 嵌套引用
body:
  greeting: "Hello ${username}, your token is ${token}"

# 在 headers、params、body 中都可使用
headers:
  Authorization: "Bearer ${access_token}"
params:
  search: "${keyword}"
```

### 步骤级变量

```yaml
steps:
  - name: "使用步骤变量"
    variables:
      local_var: "仅本步骤可见"
      api_version: "v2"
    request:
      method: GET
      url: "/api/${api_version}/users"
```

## 2. 内置动态函数

通过 `${函数名()}` 语法调用，每次执行自动生成新值：

### 基础函数

| 函数 | 说明 | 示例输出 |
|------|------|---------|
| `${timestamp()}` | Unix 时间戳（秒） | `1710057600` |
| `${timestamp_ms()}` | Unix 时间戳（毫秒） | `1710057600123` |
| `${uuid()}` | UUID v4 | `a1b2c3d4-e5f6-...` |
| `${now()}` | 当前时间 | `2026-03-10 12:00:00` |
| `${today()}` | 今日日期 | `2026-03-10` |

### 随机数据函数

| 函数 | 说明 | 示例输出 |
|------|------|---------|
| `${random_int()}` | 随机整数 (0-9999) | `4237` |
| `${random_float()}` | 随机浮点数 | `3.14` |
| `${random_string()}` | 随机字符串 (8位) | `aB3xK9mP` |
| `${random_phone()}` | 随机手机号 | `13812345678` |
| `${random_email()}` | 随机邮箱 | `abc@example.com` |
| `${random_name()}` | 随机中文姓名 | `张三` |
| `${random_id_card()}` | 随机身份证号 | `110101199001011234` |
| `${random_ip()}` | 随机 IP | `192.168.1.100` |
| `${random_url()}` | 随机 URL | `https://example.com/path` |

### 加密函数

| 函数 | 说明 | 用法 |
|------|------|------|
| `${md5(value)}` | MD5 哈希 | `${md5(password123)}` |
| `${sha256(value)}` | SHA256 哈希 | `${sha256(secret)}` |
| `${base64_encode(value)}` | Base64 编码 | `${base64_encode(hello)}` |
| `${base64_decode(value)}` | Base64 解码 | `${base64_decode(aGVsbG8=)}` |

### 使用示例

```yaml
name: "动态变量演示"
variables:
  order_no: "ORD-${timestamp()}-${random_int()}"
  trace_id: "${uuid()}"

steps:
  - name: "创建订单"
    request:
      method: POST
      url: "/api/orders"
      headers:
        X-Trace-Id: "${trace_id}"
      body:
        order_no: "${order_no}"
        user_phone: "${random_phone()}"
        user_email: "${random_email()}"
        amount: ${random_float()}
        sign: "${md5(${order_no}secret_key)}"
```

## 3. 多步骤编排与数据流转

多步骤用例的核心是 **提取 → 传递 → 使用**：

```yaml
name: "多步骤数据流转"
base_url: "https://httpbin.org"

steps:
  # 步骤1：创建资源，提取 ID
  - id: "create"
    name: "创建资源"
    request:
      method: POST
      url: "/post"
      body:
        name: "test_resource"
        type: "demo"
    extract:
      - name: resource_url
        type: jsonpath
        expression: "$.url"
      - name: resource_name
        type: jsonpath
        expression: "$.json.name"
    asserts:
      - target: status_code
        operator: eq
        expected: 200

  # 步骤2：使用步骤1提取的数据
  - id: "verify"
    name: "验证资源"
    request:
      method: GET
      url: "/get"
      params:
        resource: "${resource_name}"
        source: "${resource_url}"
    asserts:
      - target: status_code
        operator: eq
        expected: 200
      - target: body
        expression: "$.args.resource"
        operator: eq
        expected: "${resource_name}"
```

**数据流转原理图：**
```
步骤1 执行 → 响应 → extract 提取变量 → 存入变量池
                                              ↓
步骤2 构建请求 ← 从变量池解析 ${变量} ← 变量池
```

## 4. 步骤依赖与跳过

### depends_on：步骤依赖

```yaml
steps:
  - id: "login"
    name: "登录"
    request:
      method: POST
      url: "/api/login"
      body: { username: "admin", password: "123" }
    extract:
      - name: token
        type: jsonpath
        expression: "$.token"

  - id: "get_profile"
    name: "获取用户信息"
    depends_on:
      - "login"     # 依赖 login 步骤成功后才执行
    request:
      method: GET
      url: "/api/profile"
      headers:
        Authorization: "Bearer ${token}"

  - id: "update_profile"
    name: "更新用户信息"
    depends_on:
      - "login"
      - "get_profile"  # 依赖多个步骤
    request:
      method: PUT
      url: "/api/profile"
      body: { nickname: "new_name" }
```

> 当被依赖的步骤失败时，依赖它的步骤将自动跳过并标记原因。

### skip_if：条件跳过

```yaml
steps:
  - id: "check_env"
    name: "检查环境"
    request:
      method: GET
      url: "/api/health"
    extract:
      - name: env_status
        type: jsonpath
        expression: "$.status"

  - name: "仅在生产环境执行"
    skip_if:
      variable: "env_status"
      operator: "neq"
      value: "production"
    request:
      method: POST
      url: "/api/cache/clear"
```

## 5. 分支逻辑

根据条件执行不同的分支步骤：

```yaml
steps:
  - name: "获取用户类型"
    request:
      method: GET
      url: "/api/user/type"
    extract:
      - name: user_type
        type: jsonpath
        expression: "$.data.type"

  - name: "根据用户类型走不同分支"
    branch:
      condition:
        variable: "user_type"
        operator: "eq"
        value: "vip"
      if_true:
        request:
          method: GET
          url: "/api/vip/benefits"
        asserts:
          - target: status_code
            operator: eq
            expected: 200
      if_false:
        request:
          method: GET
          url: "/api/basic/features"
        asserts:
          - target: status_code
            operator: eq
            expected: 200
```

**条件运算符：**

| 运算符 | 说明 |
|--------|------|
| `eq` | 等于 |
| `neq` | 不等于 |
| `gt` | 大于 |
| `lt` | 小于 |
| `contains` | 包含 |
| `exists` | 变量存在 |

## 6. 循环逻辑

### 固定次数循环

```yaml
steps:
  - name: "批量创建测试数据"
    loop:
      times: 5    # 循环5次
    request:
      method: POST
      url: "/api/items"
      body:
        name: "item_${loop_index}"   # loop_index 从 0 开始
        batch_id: "${uuid()}"
    asserts:
      - target: status_code
        operator: in
        expected: [200, 201]
```

### 条件循环

```yaml
steps:
  - name: "轮询直到任务完成"
    loop:
      max_iterations: 10       # 最大循环次数（防止死循环）
      condition:
        variable: "task_status"
        operator: "neq"
        value: "completed"
      interval: 2              # 每次间隔2秒
    request:
      method: GET
      url: "/api/tasks/${task_id}"
    extract:
      - name: task_status
        type: jsonpath
        expression: "$.data.status"
    asserts:
      - target: status_code
        operator: eq
        expected: 200
```

### 数据驱动循环

```yaml
steps:
  - name: "遍历测试数据"
    loop:
      items:
        - { username: "user1", role: "admin" }
        - { username: "user2", role: "editor" }
        - { username: "user3", role: "viewer" }
    request:
      method: POST
      url: "/api/users"
      body:
        username: "${loop_item.username}"
        role: "${loop_item.role}"
```

## 7. 重试机制

```yaml
steps:
  - name: "不稳定接口带重试"
    retry:
      max_retries: 3        # 最多重试3次
      retry_interval: 1     # 每次间隔1秒
      retry_on_status:      # 指定哪些状态码触发重试
        - 500
        - 502
        - 503
    request:
      method: GET
      url: "/api/flaky-endpoint"
    asserts:
      - target: status_code
        operator: eq
        expected: 200
```

## 8. 自定义脚本

### 前置脚本 (setup_script)

```yaml
steps:
  - name: "带前置脚本的步骤"
    setup_script: |
      import time
      variables.set_step_var("request_time", str(int(time.time())))
      variables.set_step_var("sign", "computed_signature")
    request:
      method: POST
      url: "/api/data"
      body:
        timestamp: "${request_time}"
        sign: "${sign}"
```

### 后置脚本 (teardown_script)

```yaml
steps:
  - name: "带后置脚本的步骤"
    request:
      method: POST
      url: "/api/create"
    teardown_script: |
      # result 是当前步骤的执行结果
      if result.status_code == 201:
          variables.set_extract_var("created", "true")
```

### 自定义断言脚本

```yaml
asserts:
  - target: custom
    script: |
      # 可用变量：body (响应体), status_code, headers, response_time
      items = body.get("data", {}).get("items", [])
      result = all(item.get("price", 0) > 0 for item in items)
      message = "所有商品价格应大于0"
```

## 9. 插件系统

### 编写自定义插件

```python
# plugins/my_auth_plugin.py
from smartapi.plugins.base import HookPlugin, PluginMeta, PluginType

class ApiSignPlugin(HookPlugin):
    """自动为请求添加签名"""
    meta = PluginMeta(
        name="api_sign",
        version="1.0.0",
        plugin_type=PluginType.HOOK,
        description="自动计算 API 签名",
    )

    def activate(self, context):
        self.secret = context.get("sign_secret", "default_secret")

    def on_request(self, request_kwargs):
        import hashlib, time
        ts = str(int(time.time()))
        raw = f"{ts}{self.secret}"
        sign = hashlib.md5(raw.encode()).hexdigest()
        request_kwargs.setdefault("headers", {})
        request_kwargs["headers"]["X-Timestamp"] = ts
        request_kwargs["headers"]["X-Sign"] = sign
        return request_kwargs
```

### 在 conftest.py 中注册

```python
# conftest.py
from plugins.my_auth_plugin import ApiSignPlugin

@pytest.fixture(autouse=True)
def setup_plugins(plugin_manager):
    plugin = ApiSignPlugin()
    plugin.activate({"sign_secret": "my_secret_key"})
    plugin_manager.register(plugin)
```

## 10. 敏感信息加密

### 设置加密密钥

```bash
# 方式一：环境变量（推荐 CI/CD）
export SMARTAPI_SECRET_KEY="your_secret_key_here"

# 方式二：密钥文件
echo "your_secret_key_here" > ~/.smartapi_key
chmod 600 ~/.smartapi_key
```

### 加密敏感字段

```python
from smartapi.core.security import SecretManager

sm = SecretManager(key="your_secret_key_here")

# 加密
encrypted = sm.encrypt("my_api_token_123")
print(encrypted)  # ENC(gAAAAABm...)

# 解密
plaintext = sm.decrypt(encrypted)
print(plaintext)  # my_api_token_123
```

### 在 YAML 中使用加密值

```yaml
# environments/prod.yaml
name: prod
base_url: "https://api.example.com"
variables:
  api_token: "ENC(gAAAAABm...)"     # 自动解密
  db_password: "ENC(gAAAAABn...)"   # 自动解密
```
