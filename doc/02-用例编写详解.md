## 1. 用例文件结构

SmartAPI-Test 的用例使用 YAML 或 JSON 格式编写，结构如下：

```yaml
# 用例元信息
name: "用例名称"            # 必填
description: "用例描述"     # 选填
base_url: "https://api.example.com"  # 接口基础地址
tags:                       # 标签（用于筛选）
  - smoke
  - regression
priority: "high"            # 优先级：high / medium / low
variables:                  # 用例级变量
  username: "admin"
  timeout: 5000

# 测试步骤
steps:
  - name: "步骤1"
    # ... 步骤配置
  - name: "步骤2"
    # ... 步骤配置
```

## 2. 请求配置详解

### 2.1 HTTP 方法

支持所有标准 HTTP 方法：

```yaml
steps:
  - name: "GET 请求"
    request:
      method: GET
      url: "/api/users"

  - name: "POST 请求"
    request:
      method: POST
      url: "/api/users"

  - name: "PUT 请求"
    request:
      method: PUT
      url: "/api/users/1"

  - name: "DELETE 请求"
    request:
      method: DELETE
      url: "/api/users/1"

  - name: "PATCH 请求"
    request:
      method: PATCH
      url: "/api/users/1"
```

### 2.2 URL 参数 (Query String)

```yaml
request:
  method: GET
  url: "/api/users"
  params:
    page: 1
    size: 20
    keyword: "admin"
    status: "active"
# 实际请求：GET /api/users?page=1&size=20&keyword=admin&status=active
```

### 2.3 请求头

```yaml
request:
  method: POST
  url: "/api/data"
  headers:
    Content-Type: "application/json"
    Authorization: "Bearer my_token"
    X-Request-ID: "${uuid()}"
    Accept-Language: "zh-CN"
```

### 2.4 请求体 (JSON)

```yaml
request:
  method: POST
  url: "/api/users"
  headers:
    Content-Type: "application/json"
  body:
    username: "new_user"
    email: "user@example.com"
    age: 25
    roles:
      - "user"
      - "editor"
    profile:
      nickname: "小明"
      avatar: "https://example.com/avatar.png"
```

### 2.5 表单数据

```yaml
request:
  method: POST
  url: "/api/upload"
  headers:
    Content-Type: "application/x-www-form-urlencoded"
  body:
    username: "admin"
    password: "123456"
```

### 2.6 超时与重试

```yaml
steps:
  - name: "带超时和重试的请求"
    request:
      method: GET
      url: "/api/slow-endpoint"
    timeout: 10           # 超时时间（秒）
    retry:
      max_retries: 3      # 最大重试次数
      retry_interval: 2   # 重试间隔（秒）
```

## 3. 断言系统详解

### 3.1 断言目标 (target)

| 目标 | 说明 | 示例 |
|------|------|------|
| `status_code` | HTTP 状态码 | `200`, `404` |
| `body` | 响应体（JSON） | 通过 JSONPath 表达式取值 |
| `header` | 响应头 | 通过 expression 指定头名 |
| `response_time` | 响应时间（ms） | 毫秒数 |
| `custom` | 自定义脚本 | Python 表达式 |

### 3.2 运算符全表 (operator)

| 运算符 | 说明 | expected 示例 |
|--------|------|-------------|
| `eq` | 等于 | `200` |
| `neq` | 不等于 | `500` |
| `gt` | 大于 | `0` |
| `gte` | 大于等于 | `1` |
| `lt` | 小于 | `5000` |
| `lte` | 小于等于 | `100` |
| `contains` | 包含子串 | `"success"` |
| `not_contains` | 不包含子串 | `"error"` |
| `starts_with` | 以...开头 | `"https"` |
| `ends_with` | 以...结尾 | `".json"` |
| `regex` | 正则匹配 | `"^\\d{4}-\\d{2}"` |
| `in` | 值在列表中 | `[200, 201, 204]` |
| `not_in` | 值不在列表中 | `[500, 502, 503]` |
| `is_null` | 值为 null | 不需要 expected |
| `is_not_null` | 值非 null | 不需要 expected |
| `type_is` | 类型检查 | `"int"`, `"str"`, `"list"`, `"dict"`, `"bool"`, `"float"` |
| `length_eq` | 长度等于 | `3` |
| `length_gt` | 长度大于 | `0` |
| `length_lt` | 长度小于 | `100` |

### 3.3 断言级别 (level)

```yaml
asserts:
  # 默认级别：ERROR - 断言失败则用例失败
  - target: status_code
    operator: eq
    expected: 200

  # WARNING 级别：仅记录警告，不影响用例结果
  - target: response_time
    operator: lt
    expected: 100
    level: warning
    message: "响应时间超过 100ms，注意性能"

  # FATAL 级别：断言失败则中断整个测试集
  - target: status_code
    operator: neq
    expected: 503
    level: fatal
    message: "服务不可用，终止测试"
```

### 3.4 断言示例大全

```yaml
asserts:
  # ── 状态码 ──
  - target: status_code
    operator: eq
    expected: 200

  - target: status_code
    operator: in
    expected: [200, 201]

  # ── 响应体 (JSONPath) ──
  - target: body
    expression: "$.code"
    operator: eq
    expected: 0

  - target: body
    expression: "$.data.list"
    operator: length_gt
    expected: 0

  - target: body
    expression: "$.data.username"
    operator: eq
    expected: "admin"

  - target: body
    expression: "$.data.email"
    operator: regex
    expected: "^[\\w.]+@[\\w]+\\.[a-z]{2,}$"

  - target: body
    expression: "$.data.tags"
    operator: type_is
    expected: "list"

  - target: body
    expression: "$.data.deleted_at"
    operator: is_null

  # ── 响应头 ──
  - target: header
    expression: "Content-Type"
    operator: contains
    expected: "application/json"

  - target: header
    expression: "X-Request-Id"
    operator: is_not_null

  # ── 响应时间 ──
  - target: response_time
    operator: lt
    expected: 3000

  # ── 自定义脚本 ──
  - target: custom
    script: |
      data = body.get("data", {})
      result = data.get("total", 0) == len(data.get("list", []))
      message = "total 字段与列表长度不一致"
```

## 4. 数据提取

从响应中提取数据，供后续步骤使用：

```yaml
steps:
  - name: "登录获取 Token"
    request:
      method: POST
      url: "/api/login"
      body:
        username: "admin"
        password: "123456"
    extract:
      # JSONPath 提取
      - name: token
        type: jsonpath
        expression: "$.data.token"

      # JSONPath 提取（带默认值）
      - name: user_id
        type: jsonpath
        expression: "$.data.user_id"
        default: "0"

      # 正则提取
      - name: session_id
        type: regex
        expression: "session_id=(\\w+)"

      # Header 提取
      - name: request_id
        type: header
        expression: "X-Request-Id"
    asserts:
      - target: status_code
        operator: eq
        expected: 200

  - name: "使用提取的 Token"
    request:
      method: GET
      url: "/api/user/info"
      headers:
        # 用 ${变量名} 引用提取的值
        Authorization: "Bearer ${token}"
    asserts:
      - target: body
        expression: "$.data.id"
        operator: eq
        expected: "${user_id}"
```

**提取类型对照表：**

| 类型 | 说明 | expression 示例 |
|------|------|----------------|
| `jsonpath` | JSONPath 表达式 | `$.data.token` |
| `xpath` | XPath（XML/HTML） | `//div[@class='result']/text()` |
| `regex` | 正则表达式（取第一个分组） | `"token":"(\w+)"` |
| `header` | 响应头 | `Set-Cookie` |

## 5. 测试套件

将多个用例组织为一个测试集：

```yaml
# testsuite.yaml
name: "用户模块测试套件"
description: "覆盖用户增删改查"
variables:
  base_url: "https://api.example.com"
  admin_user: "admin"

test_cases:
  - name: "创建用户"
    steps:
      - name: "POST 创建"
        request:
          method: POST
          url: "${base_url}/api/users"
          body:
            username: "test_user"
        asserts:
          - target: status_code
            operator: eq
            expected: 201

  - name: "查询用户"
    steps:
      - name: "GET 查询"
        request:
          method: GET
          url: "${base_url}/api/users"
        asserts:
          - target: status_code
            operator: eq
            expected: 200
```

## 6. 环境配置

创建 `environments/dev.yaml`：

```yaml
name: dev
base_url: "https://dev-api.example.com"
variables:
  admin_user: "admin"
  admin_pass: "dev_password"
  timeout: 10
headers:
  X-Env: "development"
```

创建 `environments/prod.yaml`：

```yaml
name: prod
base_url: "https://api.example.com"
variables:
  admin_user: "admin"
  admin_pass: "ENC(gAAAAABm...)"  # 加密存储
  timeout: 30
headers:
  X-Env: "production"
```

**使用方式：**
```bash
# CLI 指定环境
smartapi run testcases/ --env dev

# pytest 指定环境
pytest testcases/ --smartapi-env dev
```

## 7. 完整示例

一个包含多步骤、提取、变量的完整用例：

```yaml
name: "用户注册→登录→查询完整流程"
base_url: "https://httpbin.org"
tags:
  - integration
  - user_flow
variables:
  test_user: "smartapi_${uuid()}"
  test_email: "${random_email()}"

steps:
  - id: "register"
    name: "Step1: 模拟注册"
    request:
      method: POST
      url: "/post"
      body:
        action: "register"
        username: "${test_user}"
        email: "${test_email}"
        password: "Test@123"
    extract:
      - name: registered_user
        type: jsonpath
        expression: "$.json.username"
    asserts:
      - target: status_code
        operator: eq
        expected: 200
      - target: body
        expression: "$.json.action"
        operator: eq
        expected: "register"

  - id: "login"
    name: "Step2: 模拟登录"
    depends_on:
      - "register"
    request:
      method: POST
      url: "/post"
      body:
        action: "login"
        username: "${registered_user}"
        password: "Test@123"
    extract:
      - name: login_token
        type: jsonpath
        expression: "$.json.username"
    asserts:
      - target: status_code
        operator: eq
        expected: 200
      - target: body
        expression: "$.json.username"
        operator: eq
        expected: "${registered_user}"

  - id: "profile"
    name: "Step3: 查询用户信息"
    depends_on:
      - "login"
    request:
      method: GET
      url: "/get"
      params:
        token: "${login_token}"
        action: "profile"
      headers:
        Authorization: "Bearer ${login_token}"
    asserts:
      - target: status_code
        operator: eq
        expected: 200
      - target: body
        expression: "$.args.action"
        operator: eq
        expected: "profile"
      - target: response_time
        operator: lt
        expected: 5000
        level: warning
        message: "查询接口响应较慢"
```
